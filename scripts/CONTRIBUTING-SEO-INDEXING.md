# SEO + Indexing System — TWERKHUB (source-of-truth)

Last validated: 2026-04-28. 992 HTML pages, 749 video entries in sitemap.
This is the playbook for the full SEO indexing stack — sitemaps, internal linking,
discovery hubs, video schema, image schema, and crawl optimization.

## What's wired

```
robots.txt
  └─> sitemap-index.xml
        ├─> sitemap.xml          (970+ pages, hreflang for KO/JA cluster)
        ├─> sitemap-videos.xml   (749 detail pages with full video metadata)
        └─> sitemap-images.xml   (existing, manually curated)

every detail page (751 total)
  ├─ VideoObject schema (in raw HTML, not JS-injected)
  ├─ BreadcrumbList schema
  ├─ ImageObject schema (auto-injected by image-seo-engine.js with license + creator + copyrightNotice + acquireLicensePage)
  ├─ Related videos block (12 internal links: 8 same-playlist + 4 cross)
  ├─ "Keep going" footer with cross-playlist + Discord/Telegram CTAs
  ├─ GA4 tracking + video + conversion + scroll + engagement
  └─ +18 paywall (server-side) for 36 age-restricted videos

discovery hubs
  ├─ /recent.html    (200 newest cuts, mtime-sorted)
  └─ /trending.html  (full archive, 751 cards by playlist)

homepage + footer
  └─ Links to /recent.html, /trending.html, all 6 playlist HOMEs
```

## Scripts (run before every deploy)

```bash
# 1. Refresh internal linking (idempotent — replaces existing block per page)
python3 scripts/inject-related-links.py

# 2. Refresh discovery hubs from current detail pages
python3 scripts/generate-discovery-hubs.py

# 3. Auto-generate sitemap.xml + sitemap-videos.xml + sitemap-index.xml
python3 scripts/generate-sitemaps.py

# 4. Validate (must pass before commit)
node scripts/validate-pages.js   # GA4 + image-seo-engine + structure + JS syntax
node scripts/check-encoding.js   # mojibake + BOM
```

When you add a new playlist OR new detail page, run all 4 in order. They are
idempotent — running them again costs nothing.

## VideoObject schema (already on every detail page)

Generated by the detail-page builder at creation time. Lives in raw HTML
(NOT injected by JS) so Googlebot sees it on first crawl:

```json
{
  "@context": "https://schema.org",
  "@type": "VideoObject",
  "name": "<unique title>",
  "description": "<unique description, 50-160 chars>",
  "thumbnailUrl": ["https://i.ytimg.com/vi/<id>/hqdefault.jpg"],
  "uploadDate": "2026-04-20",
  "duration": "PT3M58S",
  "embedUrl": "https://www.youtube.com/embed/<id>",
  "contentUrl": "https://www.youtube.com/watch?v=<id>",
  "creator": { "@type": "Person", "name": "<creator>" },
  "publisher": { "@type": "Organization", "name": "Twerkhub", "url": "https://alexiatwerkgroup.com/" },
  "isPartOf": { "@type": "CollectionPage", "name": "<playlist label>", "url": "https://alexiatwerkgroup.com/<playlist>/" }
}
```

Each page has UNIQUE name + description + thumbnailUrl — never duplicate
across pages. Description leverages rotating intro/why/next templates from
the detail-page generator (10 × 10 × 8 = 800 unique combos).

## Internal linking engine

`scripts/inject-related-links.py` injects between markers:

```html
<!-- twk-related-block -->
<section class="twk-related" aria-label="Related videos">
  ...8 same-playlist cards...
  ...4 cross-playlist cards...
  ...nav tags to all 6 playlists + /recent.html...
</section>
<!-- /twk-related-block -->
```

- Deterministic per-page seed (sha1 of slug) so crawl paths are stable but
  varied across the archive — every detail page reaches a different
  subset, dramatically increasing graph density.
- Re-runnable: replaces existing block, never duplicates.
- All 751 detail pages are now within 3 clicks of every other page.

## Sitemap auto-generator

`scripts/generate-sitemaps.py`:
- Walks the entire repo
- Excludes utility patterns (admin, test, _*, paid-content, *-bak)
- Skips pages with `<meta name="robots" content="...noindex...">`
- Auto-detects KO/JA cluster (hottest-cosplay-fancam + korean-girls-kpop-twerk)
  and emits `xhtml:link` hreflang annotations on EN + KO + JA versions
- Parses VideoObject JSON-LD from each detail page to populate
  sitemap-videos.xml with thumbnail_loc, title, description, content_loc,
  player_loc, publication_date, duration (in seconds), uploader

Output:
- sitemap.xml (970+ URLs)
- sitemap-videos.xml (749 video entries)
- sitemap-index.xml (master index — points to all 3)

## ImageObject extension (already in image-seo-engine.js)

The auto-injected JSON-LD includes the 4 required fields for image rich results:

```json
{
  "@type": "ImageObject",
  "contentUrl": "...",
  "url": "...",
  "caption": "...",
  "creator": { "@type": "Person", "name": "Alexia Twerk" },
  "creditText": "TwerkHub / Alexia Twerk",
  "copyrightNotice": "© Alexia Twerk",
  "license": "https://alexiatwerkgroup.com/terms",
  "acquireLicensePage": "https://alexiatwerkgroup.com/contact",
  "encodingFormat": "<auto-detected mime>"
}
```

Validator enforces presence of `image-seo-engine.js` script tag on every
non-utility page — no future page can ship without it.

## Discovery hubs

- `/recent.html` — 200 newest detail pages by mtime, single-grid view, GA4 tracked
- `/trending.html` — full archive (751 cards) grouped by playlist, sectioned
- Both are statically generated, contain CollectionPage JSON-LD,
  and are linked from homepage footer (Platform column + bottom legal row)

These give Google a single short crawl path to every page in the archive —
homepage → /trending.html → any detail page = 2 clicks max.

Google polls the sitemap on its own schedule.

## Crawl + performance

- All internal links are in raw HTML (`<a href>`) — no JS-only navigation
- Image lazy-load via `loading="lazy"` on related-cards and discovery hubs
- VideoObject schema is in raw HTML — Googlebot sees it on first byte
- No `noindex` on commercial pages (membership.html, top-100-twerk-videos.html)
- max-image-preview:large + max-video-preview:-1 + max-snippet:-1 on every detail page

## When to re-run the pipeline

| Trigger | Run |
|---------|-----|
| Added a new detail page | inject-related-links + generate-discovery-hubs + generate-sitemaps |
| Added a new playlist | Update PLAYLISTS dict in inject-related-links.py + generate-discovery-hubs.py, then run all 3 |
| Added KO/JA versions | Update I18N_CLUSTER in generate-sitemaps.py + run it |
| Updated 1+ existing pages | inject-related-links + generate-sitemaps (skip discovery if no new files) |
| Quarterly health check | Run all 3 + validator + encoding-check |

## Known gotchas

- **OneDrive sync truncation**: when editing index.html via Windows file tools,
  occasionally the trailing `<script>` block gets truncated. Run validator
  after every edit; if it complains "missing image-seo-engine script tag" or
  "doesn't end with </body></html>", restore from git HEAD and re-apply.
- **Cloudflare Pages first-deploy gotcha**: new top-level directories may be
  silently skipped. Push a follow-up trivial commit to force rebuild.
- **Sitemap-videos size limit**: Google caps single sitemap at 50,000 URLs / 50 MB.
  At current rate (749 entries → 397 KB), we won't hit either before 30,000+ pages.
  When approaching, split by playlist (sitemap-videos-cosplay.xml etc.).
